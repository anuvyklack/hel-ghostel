* hel-ghostel

Integration between [[https://github.com/anuvyklack/hel][Hel]] (Helix Emulation Layer) and [[https://github.com/dakra/ghostel][Ghostel]] (libghostty-powered
terminal emulator).

Hel provides a Helix / Kakoune editing model inside Emacs. Ghostel runs a real
terminal inside an Emacs buffer. This package wires the two together.

** Install

#+begin_src emacs-lisp
(use-package hel-ghostel
  :after (ghostel hel))
#+end_src

** Workflow

~Ghostel~ terminal emulator always starts in ~semi-char~ mode (it is hard-coded), in
which most keys are sent to the terminal (except to ~ghostel-keymap-exceptions~).
From here:

- =C-c Esc= ::
  - Switch to read-only mode with the terminal still running (~ghostel-emacs-mode~).
    Keys are not sent to the terminal, you can move around the buffer, select
    and copy any text.
  - With =M-u= — freezes the terminal (live output is paused) and switch to
    read-only mode (~ghostel-copy-mode~).

In read-only mode:
- =i= or =a= — switch to ~semi-char~ mode
- =I= or =A= — switch to ~line~ mode

In ~semi-char~ and ~line~ modes:
- =C-c i=   — switch to ~semi-char~ mode.
- =C-c I=   — switch to ~line~ mode.
- =C-c p=   — paste to terminal.
- =C-c C-p= — cycle through the ~kill-ring~ and paste
- =C-c C-w= — hel window keymap — the same an on =C-w= in normal state.

** Keybindings
*** Semi-char mode

| Key     | Command / note                              |
|---------+---------------------------------------------|
| =C-c Esc= | switch to read-only mode                    |
| =C-S-v=   | paste                                       |
| =C-c p=   | paste                                       |
| =C-c C-p= | cycle through the ~kill-ring~                 |
| =C-c I=   | switch to ~line~ mode                         |
| =C-c w=   | switch to another window in cyclic ordering |
| =C-c C-w= | hel window keymap (=C-w= in normal state)     |

*** Line mode

| Key       | Command                  |
|-----------+--------------------------|
| =C-c Esc=   | switch to read-only mode |
| =C-c i=     | switch to ~semi-char~ mode |
| =C-j= / =C-k= | history                  |

*** Read-only modes

| Key     | Command                   |
|---------+---------------------------|
| =i=, =a=    | switch to ~semi-char~ mode  |
| =I=, =A=    | switch to ~line~ mode       |
| =]]= / =[[= | *next* / *previous* prompt    |
| =]l= / =[l= | *next* / *previous* hyperlink |

** zsh-helix-mode integration

On each new shell spawn (~ghostel-pre-spawn-hook~), the following
environment variables are exported from the current Emacs theme so
that [[https://github.com/Multirious/zsh-helix-mode][zsh-helix-mode]] matches Emacs colors:

- =ZHM_STYLE_SELECTION= — foreground / background from ~region~ face
- =ZHM_STYLE_CURSOR_NORMAL= — foreground / background from ~cursor~ face
- =ZHM_CURSOR_NORMAL= — raw escape sequence for the block cursor
